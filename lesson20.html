Understanding Database Relationships

Databases use relationships to connect different tables, making data retrieval more efficient and meaningful.

Types of Relationships

  • One-to-One: Each record in Table A is related to one record in Table B.
  • One-to-Many: A single record in Table A relates to multiple records in Table B.
  • Many-to-Many: Multiple records in Table A relate to multiple records in Table B (requires a junction table).

Using JOINs to Query Related Data

SQL provides different types of JOINs to combine data from multiple tables.

Example: INNER JOIN

                
SELECT users.name, orders.amount 
FROM users
INNER JOIN orders ON users.id = orders.user_id;

Other Types of JOINs

  • LEFT JOIN: Retrieves all records from the left table and matching records from the right table.
  • RIGHT JOIN: Retrieves all records from the right table and matching records from the left table.
  • FULL JOIN: Retrieves all records when there is a match in either table.
