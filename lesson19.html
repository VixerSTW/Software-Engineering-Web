What is a Database Schema?

A database schema defines the structure of a database, including tables, columns, data types, and relationships.

Key Concepts of Database Schema Design

  • Tables: Store data in rows and columns.
  • Primary Keys: Unique identifiers for each record.
  • Foreign Keys: Links between tables to enforce relationships.
  • Normalization: Organizing data to reduce redundancy.

Example Database Schema

                
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100) UNIQUE
);

CREATE TABLE orders (
    id INT PRIMARY KEY,
    user_id INT,
    amount DECIMAL(10,2),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
